C. Seguridad

Lo que está bien

  • ✓ Headers: X-Frame-Options DENY, X-Content-Type-Options nosniff, Referrer-Policy, Permissions-Policy
  • ✓ Validación server-side: regex email/teléfono, límite de caracteres
  • ✓ escapeHtml() para prevenir XSS en emails
  • ✓ Rate limiting (5 req/min por IP)
  • ✓ .env en .gitignore con .env.example sin secretos
  • ✓ rel="noopener noreferrer" en todos los links externos

Pendiente

PrioridadProblemaDetalle
ALTASin Content-Security-Policy (CSP)El sitio carga scripts de googletagmanager.com, connect.facebook.net y usa inline scripts. Un CSP protegería contra scripts inyectados.
ALTARate limiter es por instancia serverlessrateMap es un Map() en memoria. En Vercel cada invocación puede ser una instancia nueva, haciendo el rate limiting inefectivo. Usar Upstash Redis o Vercel KV.
ALTASin protección CSRFCualquier sitio puede hacer POST a /api/lead. Agregar verificación de Origin header.
MEDIAFalta header Strict-Transport-SecurityAgregar HSTS para forzar HTTPS.
MEDIAAPI devuelve el lead completoreturn { ok: true, lead } expone datos internos (timestamp, origen). Devolver solo { ok: true }.
MEDIAIDs de GA4 y Pixel hardcodeadosMover a variables de entorno NEXT_PUBLIC_GA_ID y NEXT_PUBLIC_FB_PIXEL_ID.
BAJASin CAPTCHA en formulariosBots pueden enviar leads. Agregar reCAPTCHA v3 o Cloudflare Turnstile (requiere dominio propio).
BAJACampo modelo sin validar contra valores permitidosUn atacante puede enviar nombres de modelo arbitrarios. Validar contra ['MG3 Hybrid+', 'MG ZS Hybrid+'].

Lo que está bien

  • ✓ Labels asociados a inputs con htmlFor

Pendiente

PrioridadProblemaDetalle
ALTAHero dots sin aria-label<button className="hero-dot"> vacío — lectores de pantalla ven botones en blanco. Agregar aria-label={`Slide ${i+1}`}.
ALTABotón cerrar modal sin aria-labelEl × de ModelModal y Lightbox no tiene aria-label="Cerrar".
ALTADots de colores son <div> no <button><div className="modal-color-dot" onClick> no es focusable por teclado, sin role ni aria-label.
ALTAModel cards usan <div onClick>No son focusables por teclado. Tab no los alcanza, Enter no los activa. Usar <button> o <a>.
MEDIAFAQ sin aria-expandedLos botones del acordeón no indican estado abierto/cerrado. Agregar aria-expanded.
MEDIAModal no atrapa el focoAl abrir ModelModal, el usuario puede Tab a elementos detrás del modal. Implementar focus trap.
MEDIASin "Skip to content"No hay forma de saltar la navbar por teclado.
MEDIASVGs decorativos sin aria-hiddenSVGs de iconos (shield, map, phone, etc.) generan ruido en lectores de pantalla. Agregar aria-hidden="true".
BAJAContraste insuficiente en testimonios.testimonial-author span usa #999 sobre #f5f5f5 = ratio 2.85:1 (falla WCAG AA).
BAJASin prefers-reduced-motionMuchas animaciones (hero zoom, scroll reveal, parallax) sin acomodación para usuarios sensibles al movimiento.

PrioridadProblemaDetalle
ALTASin monitoreo de erroresSin Sentry, LogRocket ni Vercel Analytics. Si la API falla o el sitio crashea, nadie se entera. Agregar Sentry (plan gratuito).
ALTASin dominio propioEl sitio corre en giama-mg.vercel.app. Para un negocio real necesita dominio propio (giamamg.com o giamamg.com.ar).
